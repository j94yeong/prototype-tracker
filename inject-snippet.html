First-Click Tracker

A local-only, no-cloud first-click test harness for HTML prototypes.

Quick Setup (3 steps)

1Copy the tracker files alongside your prototype

Your prototype folder should look like this:

my-prototype/ your-prototype.html <-- your existing file lib/ sha256.js canonical.js html2canvas.min.js tracker.js

Copy the lib/ folder and tracker.js from the tracker package into your prototype folder.

2Add this snippet before </body> in your prototype HTML
<!-- First-Click Tracker -->
<script src="lib/sha256.js"></script>
<script src="lib/canonical.js"></script>
<script src="lib/html2canvas.min.js"></script>
<script src="tracker.js"></script>
<!-- End First-Click Tracker -->
Note: The paths assume tracker.js and lib/ sit in the same folder as your HTML file. Adjust if your folder structure differs.
3Zip it and send to testers
  1. Zip the entire my-prototype/ folder.
  2. Send the zip to your tester.
  3. They unzip it and double-click your-prototype.html to open it in their browser.
  4. They see a "First-Click Test" overlay — they enter their name (optional) and click Start.
  5. They click the first thing they'd naturally interact with.
  6. A session_XXXXXXXX.fct file downloads automatically.
  7. They send the .fct file back to you.

Reviewing Results

Open viewer.html in your browser
  1. Double-click viewer.html (it's in the tracker package root).
  2. Drag and drop all the .fct files you received from testers onto the viewer, or use "Load .fct files".
  3. Each session appears in the sidebar with its tester name, time-to-click, and a validity flag.
  4. The screenshot shows hotspot rings where each tester clicked.
  5. Hover a hotspot to see tester name, element, text, and time-to-click.
  6. Click a session in the sidebar (or a hotspot) to highlight it.
Validity flags: ✓ Valid means the file's signature matches. ⚠ Signature mismatch means the file was modified after capture (or opened from a different tracker version). These are advisory — you decide whether to trust the session.

Notes

  • Everything runs locally — no data leaves the tester's machine except the .fct file they send you.
  • Works offline. No internet connection required.
  • The tracker captures a screenshot of the page before the tester clicks, so you see exactly what they saw.
  • If multiple files come from the same tester name, the viewer keeps only the earliest attempt.
